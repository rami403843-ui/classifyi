# Инструкция для Эльдара 🙂

## 1. Запуск на своём компьютере
1. Распакуй папку `classifyi` куда удобно.
2. Открой терминал в этой папке.
3. По очереди:
```
pip install -r requirements.txt
python manage.py runserver
```
4. Открой в браузере: http://127.0.0.1:8000
5. Вход: логин `test`, пароль `test12345` (или зарегистрируй нового).

## 2. Как выложить на GitHub (с историей коммитов!)
Учитель хочет видеть НЕСКОЛЬКО коммитов, не один. Делай так:

1. Создай на github.com новый пустой репозиторий `classifyi`.
2. В терминале в папке проекта:
```
git init
git add manage.py classifyi/ requirements.txt .gitignore
git commit -m "Старт проекта: настройка Django"

git add ads/models.py ads/migrations/
git commit -m "Модели Ad и AdImage"

git add ads/views.py ads/forms.py ads/urls.py ads/admin.py
git commit -m "Views: список, фильтры, CRUD, регистрация"

git add ads/templates/ ads/static/
git commit -m "Шаблоны и стили"

git add .
git commit -m "README и финальные правки"

git branch -M main
git remote add origin https://github.com/ТВОЙ_ЛОГИН/classifyi.git
git push -u origin main
```

## 3. Скриншоты (3–5 штук)
Сделай скрины: главная с объявлениями, детальная страница,
форма создания, поиск с фильтром, «Мои объявления».
Вставь их в README.md.

## 4. Шпаргалка для защиты (3–5 минут)
1. **Идея**: «ClassiFyi — доска объявлений, как мини-lalafo. Пользователь
   регистрируется, выкладывает товар с фото и ценой».
2. **Главная**: показать каталог карточек. «Показываются только активные
   объявления, проданные скрываются».
3. **CRUD**: войти как test → «Подать объявление» → создать с фото →
   отредактировать → показать, что чужое редактировать нельзя.
4. **Поиск/фильтр**: вбить слово в поиск, выбрать категорию, цену от/до.
   «Фильтры читаются из GET-параметров и применяются через filter() в Django ORM».
5. **Фишки**: несколько фото, статус продано, счётчик просмотров
   (обновить страницу — число растёт), «Мои объявления», адаптив (сузить окно).
6. **Самое сложное**: «Несколько фото на объявление — сделал отдельную
   модель AdImage со связью ForeignKey, и загрузку через request.FILES.getlist».
7. **В будущем**: лайки, избранное, чат покупателя с продавцом.

## Как устроен код (если спросят)
- **models.py** — 2 модели: `Ad` (объявление) и `AdImage` (фото, связано с Ad).
- **views.py** — функции-обработчики: список с фильтрами, детальная (+1 просмотр),
  создание/редактирование/удаление (@login_required + проверка автора), регистрация.
- **urls.py** — какие адреса какой функции соответствуют.
- **templates/** — HTML-страницы, все наследуются от base.html.
- **static/css/style.css** — все стили, свои, без Bootstrap.
